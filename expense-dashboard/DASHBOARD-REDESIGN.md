# AS3 Expense Dashboard - Redesign Specification

## Core Philosophy

**"Report by Exception, Decide with Context, Learn from Corrections"**

This is NOT an expense management app. It's a **Human Oversight Console** for an automated system.

### The Two Jobs of This App

1. **FEED THE SYSTEM** - Import bank transactions (CSV uploads from AMEX/Wells Fargo)
2. **HANDLE EXCEPTIONS** - Review items that automation couldn't confidently process

## Interaction Flows

### Flow 1: Approve Reimbursement (Happy Path)
1. User sees reimbursement card at top (highest priority)
2. Glances at receipt thumbnail, amount looks right
3. Clicks "Pay via Payroll"
4. Card animates out, count decrements
5. Toast: "Reimbursement approved - added to payroll batch"

### Flow 2: Approve Orphan (Happy Path)
1. User sees orphan card with AI suggestions
2. State: CA (parsed), Category: Fuel-COS (vendor rule) - both look correct
3. Clicks "Approve & Post"
4. Card animates out
5. Toast: "Posted to QBO - Fuel COS, CA"

### Flow 3: Correct an Orphan
1. User sees orphan card, state is wrong (AI said CA but it's TX)
2. Clicks "Edit"
3. Edit panel expands inline
4. Changes state to TX
5. Checks "Create vendor rule" checkbox
6. Clicks "Save & Post"
7. Toast: "Posted to QBO - Rule created for CHEVRON patterns in TX"

### Flow 4: Reject Expense
1. User sees expense that looks fraudulent/invalid
2. Clicks "Reject"
3. Confirmation modal: "Are you sure? This cannot be undone."
4. Confirms
5. Card removed, expense marked rejected

## Data Requirements

### System Health Query
```sql
-- Auto rate this week
SELECT
  COUNT(*) FILTER (WHERE status IN ('matched', 'orphan_processed')) * 100.0 /
  COUNT(*) as auto_rate
FROM bank_transactions
WHERE created_at > NOW() - INTERVAL '7 days';

-- Bank feed freshness
SELECT MAX(created_at) as last_import FROM bank_transactions;

-- n8n status (external API call or stored in DB)
```

### Attention Items Query
```sql
-- All items needing attention, ordered by priority
SELECT
  'reimbursement' as type,
  eq.*,
  EXTRACT(EPOCH FROM (NOW() - eq.created_at))/86400 as days_waiting
FROM expense_queue eq
WHERE eq.is_reimbursement = true AND eq.status = 'pending'

UNION ALL

SELECT
  'orphan' as type,
  bt.*,
  EXTRACT(EPOCH FROM (NOW() - bt.transaction_date))/86400 as days_old
FROM bank_transactions bt
WHERE bt.status = 'unmatched'
  AND bt.transaction_date < NOW() - INTERVAL '5 days'

UNION ALL

SELECT
  'low_confidence' as type,
  eq.*,
  eq.confidence_score
FROM expense_queue eq
WHERE eq.is_reimbursement = false AND eq.status = 'pending'

ORDER BY
  CASE type
    WHEN 'reimbursement' THEN 1
    WHEN 'orphan' THEN 2
    ELSE 3
  END,
  days_waiting DESC NULLS LAST;
```
